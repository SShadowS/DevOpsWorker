# Git Branch Naming Convention

| Type        | Pattern                                        |
|-------------|------------------------------------------------|
| Bug fix     | `bug/#[work-item-number]-short-description`    |
| User story  | `userstory/#[work-item-number]-short-description` |

- **Base branch:** always `master`
- **Short description:** lowercase, hyphens, derived from work item title

## Examples

- Bug #12345 "Fix email sending" --> `bug/#12345-fix-email-sending`
- User Story #67890 "Add delivery method" --> `userstory/#67890-add-delivery-method`
